## 2 · 基础架构现状

### 2.1 TvStage / scale 自适应

- **没有独立的 `TvStage.vue` 组件**，1280×720 画布通过 `App.vue` 里的 `.tv-stage` CSS 类实现
  - 位置：`src/styles/global.css` line 22-29
  - 尺寸：`width: var(--vp-w) 1280px; height: var(--vp-h) 720px`
  - 居中策略：`@media (min-width: 1281px), (min-height: 721px)` 下 body 变 flex 居中
  - **只有居中，没有 scale 自适应**。在实际 4K / 1080 TV 上能否正确缩放？依赖 Shell/WebView 行为，tv-html 自身不做 transform scale
- Gallery 用的是自己的 `.gallery-stage { width: 1280px; height: 720px }`，写死固定尺寸
- ⚠️ **TV_TASKS_v6 要求的"scale 自适应容器"实际未实现**。主控台定夺是否这次一起做

### 2.2 assets.ts（CDN 助手）

- 路径：`src/utils/assets.ts`（54 行）
- **不是映射表**，而是一个简单的前缀拼接函数：
  ```ts
  export function asset(relativePath: string): string {
    return `${activeBase}/${clean}`;
  }
  ```
- 调用方必须自己写对路径字符串，如 `asset('bear/bear_paint.webp')`
- 不存在 NAMING_CONTRACT 的运行时校验，**文件名写错只会 404**
- 支持 `setAssetBase()` 运行时换 CDN（OEM 自建镜像场景）
- 默认 base：`https://cdn.jsdelivr.net/gh/snugogo/wonderbear@main/assets`

### 2.3 焦点态 CSS（.F.fx 的实际实现）

**项目用的不是 `.F.fx` 命名，而是两套并存**：

1. **`data-focused='true'` 属性**（全局、所有 useFocusable 节点自动挂载）
   - 位置：`src/styles/global.css` line 40-50
   - 样式（含未 commit 的新 coral 环）：
     ```css
     [data-focused='true'] {
       outline: 4px solid var(--c-focus);      /* 未提交前是 --c-amber */
       outline-offset: 4px;
       box-shadow: var(--shadow-focus);        /* 三层珊瑚环 */
       transition: outline-color 120ms, box-shadow 120ms;
       z-index: 1;
       position: relative;
     }
     ```

2. **`.wb-focus-feedback` + `.is-focused` class**（组件手动切换，提供弹簧变换）
   - 位置：`src/styles/global.css` line 74-96
   - 核心规则：
     ```css
     .wb-focus-feedback {
       transition: transform 350ms cubic-bezier(0.34, 1.56, 0.64, 1), ...;
     }
     .wb-focus-feedback.is-focused { transform: scale(1.06); }
     .wb-focus-feedback .icon-floatable { ... }
     .wb-focus-feedback.is-focused .icon-floatable {
       animation: bear-float 2.8s ease-in-out infinite;
     }
     ```
   - 焦点时 scale 1.06，不带 `translateY(-4px)`（工单硬伤 5 期望的)
   - 动画缓动是弹簧 `cubic-bezier(0.34, 1.56, 0.64, 1)` 350ms，**不是**工单说的 200ms ease-out

**两套并存的后果**：
- 一个 focusable 节点**同时**挂 `data-focused` 描边环 和 `.wb-focus-feedback` 弹簧
- `data-focused` 自动由 focus store 设置；`.is-focused` 需要组件自己 onFocusChange 维护
- MenuCard / 各 Screen 的按钮都两套都挂
