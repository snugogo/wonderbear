# docs/ 文件夹索引

按用途分 3 个子目录 + 1 份变更日志,**任何新文档都按这个分类规则归位**。

## 📁 spec/ — 权威规范

代码必须遵守的规范文档,**任何接手新批次的窗口都要先读**。

| 文件 | 说明 |
|---|---|
| `PROMPT_SPEC_v7_1.md` | LLM + 生图 prompt 的权威版本(v7.1)。批次 4 写故事生成时必须按它实现 |
| `API_CONTRACT_PATCH_v2.md` | OemConfig 加 h5BaseUrl 的协议补充。给创始人贴到外部 API_CONTRACT.md 用 |

**规则**:这里的文档**不会随批次推进而过期**,长期有效。

## 📁 handoff/ — 窗口交接包

每个批次完成后,给下一个批次窗口的接手指南。

| 文件 | 说明 |
|---|---|
| `HANDOFF_BATCH2.md` | 批次 1 完成后,给批次 2 窗口的接手包 |

**规则**:每完成一个批次,本目录新增一份 `HANDOFF_BATCH{N+1}.md`。
完成后老 HANDOFF 可保留作为历史记录。

## 📁 ops/ — 操作指南

给创始人/运维操作时查阅的步骤文档。

| 文件 | 说明 |
|---|---|
| `GIT_INIT.md` | monorepo 首次初始化 + push GitHub 的命令清单 |

**未来会加**:`DEPLOY.md`(部署到云服务器步骤)、`BACKUP.md`(数据库备份)等。

## 📄 CHANGELOG.md — 变更日志

**根目录**唯一的"动态文档",**所有跨窗口协议变更、文档版本变化都记一笔**。

按时间倒序追加,最新的写在最上面。

## 归类原则速查

| 我现在要写的文档是什么? | 放哪里? |
|---|---|
| 代码必须按它实现 | `spec/` |
| 给下一个 Claude 窗口看的接手指南 | `handoff/` |
| 操作步骤(部署/初始化/备份等) | `ops/` |
| 跨窗口协议变更(发了 TO_SERVER_xxx) | 在 `CHANGELOG.md` 追加一条 |
| 临时草稿 / 个人笔记 | **不放这里**,放本地不进 git |
