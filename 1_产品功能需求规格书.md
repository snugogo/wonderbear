## 一、产品愿景与定位

### 1.1 一句话定义

> **WonderBear 是一台 AI 驱动的儿童睡前故事投影仪——孩子用蓝牙语音遥控器说话,AI 十秒内生成一本独一无二的水彩绘本,投影在床头讲给孩子听。**

### 1.2 目标用户

**最终使用者**：3-8 岁儿童（核心 4-6 岁）

**购买决策者**：25-40 岁的父母（妈妈为主），有文化、重视孩子教育、有一定消费力

**市场定位**：
- 🌍 海外优先：**波兰 → 罗马尼亚 → 英语市场（UK/US/澳新）**
- 🏠 销售渠道：海外经销商 + OEM 贴牌
- 💰 目标零售价区间：€129 - €169（早鸟 €89）

### 1.3 产品核心价值

**对孩子**：
- "我的愿望被画成了故事" → 被倾听的仪式感
- 双语听故事 + 认字 → 学习中获得乐趣
- 睡前半小时的稳定陪伴 → 情绪安抚

**对家长**：
- 解放自己念故事的时间 → 劳累一天后的喘息
- 孩子的每个故事都能导出为 PDF 绘本 → 纪念意义强
- 不依赖手机屏幕 → 规避蓝光焦虑
- 双语学习能力 → 教育投资理由

**对经销商**：
- **SKU 单一（一台硬件多语言通吃）** → 库存压力小
- **硬件一次售出 + 内容订阅复购** → 持续收入流
- **品牌可贴牌** → 本地化经营空间

## 四、L4 · TV 端功能详解

### 4.1 首启激活流程（新设备开机首次）

**目标**：经销商发货时硬件已预装系统,用户开机后经家长扫码完成激活,全程不超过 3 分钟。

#### 流程
```
1. 投影仪首次通电 → 自动连接附近开放 WiFi(如无则提示家长配置)
2. 屏幕显示全屏水彩欢迎页:
   - 左侧:"欢迎来到 WonderBear" + "和小熊一起,探索新故事"
   - 右侧:二维码(小熊从二维码后探出动画)
   - 底部:"📱 家长扫码,让小熊认识你"
3. 家长扫码 → 跳转到 H5 注册页(浏览器打开,携带 deviceId 参数)
4. 家长完成注册(手机号/邮箱 + 验证码)+ 新建第一个孩子档案
5. TV 端每 3 秒轮询激活状态,检测到已激活 → 播放"小熊第一次见面"动画
6. 跳转首页,顶部显示孩子头像+名字+金币
```

#### 关键规则
- 未激活状态下,**遥控器按键只响应 WiFi 配置和帮助,其他按键不生效**
- 激活完成后本地标记 `wb_registered = true`,以后不再出现欢迎页
- 如设备断网:显示"请检查网络"页,并提供**热点 SSID / 密码配置界面**（输入用遥控器数字键或手机 H5 远程推送）
- **3 分钟内完成注册的用户**:TV 端自动播放一段"小熊欢迎 [孩子名字]"的个性化 TTS（利用刚注册的信息,增强仪式感）

#### 异常处理
- 二维码扫描后 H5 加载失败 → TV 端显示"如扫码无反应,请在浏览器访问 wonderbear.app/[6位激活码]"（激活码来自服务端,6 位字母数字）
- 10 分钟内未完成激活 → 屏保模式（小熊睡觉动画）,遥控器任意键唤醒继续激活流程

### 4.2 首页菜单（已激活设备默认页）

#### 顶部状态条（全局组件）
- **左侧**：孩子头像（72px 圆形）+ 名字（28px 粗体）+ 金币图标+数字
- **中间**：产品 Logo(可 OEM 替换,详见文档 5)
- **右侧**：天气图标 + 时间 + "👥 切换孩子"按钮

#### 6 个核心功能入口(菜单卡片)

| 菜单 | 菜单名(中文) | 英文名 | 图标视觉 | 功能 |
|---|---|---|---|---|
| 1 | **创作小屋** | Create | `bear_studio.webp` 小熊画画 | 新建故事(核心入口) |
| 2 | **故事世界** | Stories | `bear_world_map.webp` 小熊看地图 | 浏览社区公开故事(Phase 2) |
| 3 | **我的小书屋** | My Library | `bear_reading_hero.webp` 小熊读书 | 已创作的故事列表 |
| 4 | **一起看看** | Explore | `bear_explore_together.webp` 小熊指路 | 排行榜+推荐(Phase 2) |
| 5 | **我的家** | My Home | `bear_my_home.webp` 小熊在小屋前 | 个人中心+设置 |
| 6 | **投屏播放** | Cast | `ui_player_play.webp` | 手机投屏到投影仪(Phase 2) |

#### 首页交互细节
- 菜单卡片通过遥控器方向键导航,当前焦点有金黄色描边
- **焦点卡片**：`bear_xxx` 小熊播放浮动动画(上下 ±12px,2.8s 周期)
- **非焦点卡片**：动画暂停(省 CPU)
- 焦点切换有 120ms 节流,防止快速按键导致 UI 抖动
- Enter 键 → 进入对应功能；Back/Escape 键 → 首页无操作(已是顶层)

### 4.3 创作小屋(故事生成核心流程)

**这是产品的心脏**。所有技术投入和用户体验打磨都围绕这一条链路。

#### 对话轮次设计(7 轮为标准,年龄自适应)

**为什么是 7 轮而不是 4 轮**:
- 4 轮收集的信息太少,AI 生成的故事容易干瘪
- 7 轮能收集足够细节(主角+外观+场景+冲突+转折+解决+情绪)
- 7 轮问答约 2-3 分钟,小朋友专注力刚好能坚持
- 从第 4 轮起,允许孩子按 OK 键"提前说完了",AI 用已有信息生成

#### 完整流程
```
入口(首页"创作小屋"卡片)
  ↓
① 欢迎引导页(bear_welcome 动画)
   "小熊:今天想听什么故事呀?按一下遥控器的 🎤 键告诉我"
  ↓
② 7 轮对话(标准 5-6 岁版):
   轮 1:"今晚故事的主角是谁呀?"
   轮 2:"他/她长什么样?有什么特别的?"
   轮 3:"他/她在哪里冒险呢?"
   轮 4:"路上遇到了什么呀?"
        小熊提示:"再讲讲故事会更精彩哦,不想说就按 OK 键,熊熊就开始画啦~"
   轮 5:"然后发生了什么?"
   轮 6:"他/她是怎么解决的?"
   轮 7:"故事最后的感觉是什么?"(温暖/刺激/搞笑)
  ↓
③ 生成等待页(bear_paint 动画 + 进度条)
   "小熊正在给你画故事...(已画完第 3 页 / 共 12 页)"
   耗时 60-90 秒
  ↓
④ 完成页(bear_cheer 动画 + 撒彩屑)
   "画好啦!要现在听吗?" → [立即播放] [放进书屋]
```

#### 年龄自适应轮次

| 年龄 | 轮次 | 说明 |
|---|---|---|
| 3-4 岁 | **5 轮** | 问题简化,每问只要一个简单信息(主角/场景/一件事/主角的感受/结尾) |
| 5-6 岁 | **7 轮**(标准) | 见上方流程 |
| 7-8 岁 | **7 轮 + 自由补充** | 第 7 轮后加一句"你还想告诉小熊什么细节吗?可以直接按 OK 键结束" |

#### 提前结束规则

- 从**第 4 轮**开始,每轮对话后 TV 端显示提示:"按 OK 键让熊熊开始画画"
- 孩子按 OK → 跳过剩余轮次,进入生成阶段
- 第 7 轮之后强制结束(防止孩子一直讲,AI 信息过载)
- 若孩子 7 轮后仍想继续:"熊熊先把这些画成故事,下次再继续哦~"

#### 生成中断和异常处理

- **网络中断**：自动保存到本地队列,恢复后续上传
- **生成失败**：
  - 图片失败 → 用 `story_generic_forest/sky/ocean` 之一兜底
  - LLM 失败 → 重试 3 次,仍失败则提示"小熊累了,稍等再试"并退回首页
- **内容安全**：对话内容经三级过滤(详见文档 7),Level 1 引导话题转换,Level 3 暂停流程

### 4.4 故事播放

#### 播放界面
- 全屏水彩插图(16:9 构图填满画面)
- **底部字幕区**:
  - 主语言(大字号,孩子年龄对应字号)
  - 副语言(小字号,斜体,学习语言)
  - TTS 实时高亮当前朗读的文字(识字启蒙)
- **右下角播放控制**(淡入淡出):
  - 上一页 / 播放-暂停 / 下一页 / 进度条 / 退出

#### 播放交互规则
- **自动播放模式(默认)**：每页播放完 TTS 后自动翻下一页,页间停顿 1 秒
- **手动模式(家长可在 H5 设置)**：每页需按 OK 键才翻页,用于识字学习
- 翻页有水彩渐变动画(~400ms)
- 12 页结束后跳转结尾页(bear_happy 鞠躬动画 + "故事讲完啦,晚安❤")

#### BGM 策略(分场景)
| 场景 | BGM 行为 |
|---|---|
| 封面页(第 1 页出现时) | 轻仪式感,30% 音量,5 秒淡入 |
| 第 1-11 页 TTS 播放 | BGM 20% + TTS 100%,Android 原生双轨混音 |
| 第 12 页(结尾) | BGM 淡出到 0(给孩子"故事结束"的闭合感) |
| 家长可关闭 | H5 设置"播放背景音乐"toggle,同步到 TV 端 |

#### 识字模式(差异化功能)
- 入口:故事播放中按遥控器"🔤"键(或 H5 设置默认开启)
- 效果:
  - 屏幕出现大字号字幕(60px 加粗)
  - 每个字/词播放到时,单独放大 + 变色
  - 小熊在画面角落做"bear_pointing"指字动作
  - **单字可按 OK 键重复听**
- 退出:再按一次"🔤"键或自动播放完毕

### 4.5 我的小书屋（故事库）

#### 功能
- 网格展示所有已创作的故事(封面图+标题+已下载状态)
- 排序:最新创作 / 最多播放 / 收藏
- 每个故事的操作:播放 / ⬇ 下载到本地 / ❤ 收藏 / 分享(生成二维码) / 删除
- 长按删除 → 确认弹窗

#### 存储逻辑(双层:服务器 + 设备本地)

**服务器端(所有故事元数据永久保存,图片/音频存 CDN)**:
- 免费用户:最多 20 本完整故事(达到上限时,新生成前提示删除旧故事)
- 订阅用户:无限保存

**设备本地缓存(GP15 的 8GB ROM 空间有限,约 2-3GB 可用)**:
- **自动缓存**:最近播放的故事自动缓存到本地(LRU 淘汰策略)
- **手动下载**:用户按 ⬇ 按钮主动保存到本地,即使不联网也能播放
- **收藏保护**:被 ❤ 收藏的故事**不会被自动 LRU 淘汰**
- **本地上限**:默认 50 本(约 210MB),达到上限时提示用户清理

**单本故事大小参考**(1280×720 + 双语 TTS):
- 12 张水彩图 WebP ≈ 960 KB
- 1 张封面图 ≈ 80 KB
- 12 段双语 TTS MP3 ≈ 3 MB
- 开场介绍 + 元数据 ≈ 160 KB
- **合计约 4.2 MB/本**

**空间管理 UI**:
- "我的小书屋"右上角显示:`本地 12/50 本 · 剩余 158MB`
- 达到上限时,提示"您的小书屋满了,删除几本不常看的故事吧"
- 已下载到本地的故事:封面右下角有小"⬇"标识
- 仅服务器端的故事:无标识,点播放时自动从 CDN 下载

#### 手机浏览器模拟(开发调试用)

当 HTML 在手机浏览器中运行时,**必须提供触摸→方向键模拟层**:
- 左右滑动 → 方向键左/右
- 上下滑动 → 方向键上/下

- 长按 → 返回键(Escape)
- 右下角浮动"🎤"按钮 → 模拟遥控器语音键(录音通过 `navigator.mediaDevices`)

这个模拟层不进入生产 APK,只在开发环境(`?dev=1` URL 参数)启用。

## 五、L3 · 家长 H5 功能详解

### 5.1 注册与登录

#### 注册方式:**邮箱 + 验证码**(MVP 主推,单一路径)

**为什么不用手机短信**:
- 海外短信成本高(波兰 €0.05/条,罗马尼亚 €0.06/条)
- 需要额外第三方服务(Twilio/阿里云),增加复杂度
- 海外 3-8 岁孩子的家长 90%+ 有常用邮箱,邮箱验证覆盖足够
- 邮箱验证码成本近乎为 0(Resend 免费额度 100 封/天,超出 $0.001/封)

**邮件服务商选型**:
- MVP:Resend(开发友好,免费额度大)
- 规模化:SendGrid 或 Amazon SES

**国内市场未来计划**(Phase 2):
- 加微信登录
- 国内手机号做二次选项

#### 注册流程(扫码自 TV 端而来)
```
TV 扫码 → 手机浏览器打开 h5.wonderbear.app/register?device=xxx&code=yyy
  ↓
① 填写邮箱地址
  ↓
② 服务端发 6 位数字验证码邮件(60 秒有效,3 次重发冷却)
  ↓
③ 填验证码 + 设置密码(可选,留空则只用邮箱+验证码登录)
  ↓
④ 绑定设备(deviceId + 激活码 URL 自动带过来)
  ↓
⑤ 首次进入 → 引导创建第一个孩子档案
  ↓
⑥ 跳转首页,显示"设备已绑定"
```

#### 已有账号登录
- 方式 1:邮箱 + 密码(如注册时设置了密码)
- 方式 2:邮箱 + 验证码(随时可用,主推)
- "记住我" checkbox:30 天免登录

#### 未来扩展(Phase 2+)
- 微信登录(国内市场)
- Google / Apple 社交登录
- 忘记密码重置(邮件链接)

### 5.2 孩子档案管理

- 一个家长可管理**最多 4 个孩子**(MVP 范围)
- 每个孩子独立:头像/名字/年龄/性别(可不选)/主语言/学习语言/生日
- **头像**:从 20 个预设水彩头像中选(详见文档 4)

#### UI 布局建议
- **手机 H5**:
  - 1-2 个孩子 → 横向大卡片展示

### 5.4 PDF 绘本导出

#### 规格
- A4 横版(297mm × 210mm)
- **目标打印精度**:150 DPI(家庭打印机足够),高端打印店可达 250+ DPI
- **封面 + 版权页 + 12 页正文 = 14 页**(不含封底,家长自己打印装订不需要封底)
- 双集合订本:27 页(两个故事串联,去掉中间重复的版权页)

#### 图像精度如何达到 150 DPI

150 DPI @ A4 横版 = 1754 × 1240 像素

不同生图通道的输出精度:
| 通道 | 原始分辨率 | PDF 打印质量 |
|---|---|---|
| OpenAI gpt-image-1.5 | 1536 × 1024 | 需轻度放大(1.14x),打印 130 DPI |
| Gemini Imagen 3 | 1536 × 1024 | 同上 |
| FAL Flux Kontext | 1920 × 1080 | 原生 165 DPI,完美够用 |

**服务端处理策略**:
- **默认情况**:PDF 生成时优先用 FAL 的 1920×1080 原图
- **第 1 页封面**(通常是 OpenAI/Imagen 输出):用 sharp 或 Real-ESRGAN 2x 超分到 3072×2048,打印 > 250 DPI
- **超分缓存**:超分后的高清图单独存 CDN,不影响 TV 端显示

#### 为什么 WebP 不能打印 PDF

- WebP 格式部分 PDF 阅读器 / 打印机不支持
- **解决方案**:服务器保留两套图片:
  - **PNG 原图**(~1-2MB/张):用于 PDF 嵌入
  - **WebP 压缩图**(~80KB/张):用于 TV / H5 显示
- 存储成本几乎不增加:PNG 1.5MB + WebP 80KB ≈ 1.58MB/张
- R2 免费 10GB 约可存 6000 本故事(远超 MVP 需求)

#### 每页排版
- 故事插图(16:9 横版,占页面 62% 高度)
- 主语言(22pt)
- 副语言(14pt 斜体)
- 页码(右下角)

#### 封面层次(Canvas 叠加)
- Layer 1:AI 专属封面图(不复用第 1 页插图,单独生成)
- Layer 2:底部渐变蒙版(让标题可读)
- Layer 3:书名(中英双语)
- Layer 4:作者署名徽章("✍ 作者:Zosia · 6岁")
- Layer 5:品牌 Logo(可 OEM 替换)

- 免费用户:锁定(引导升级)

## 十、关键业务规则(Business Rules)

### 10.1 免费 vs 订阅界限

| 能力 | 免费 | 月度 | 年度 |
|---|---|---|---|
| 激活设备 | ✅ | ✅ | ✅ |
| 生成故事数 | 6 本一次性(不是每月) | 无限 | 无限 |
| 故事保存数 | 20 | 无限 | 无限 |
| PDF 导出 | ❌(锁定) | 每月 5 次免费 + 超出 €0.99/次 | 同月度 |
| 优先队列 | ❌ | ✅ | ✅ |
| TTS 音色选择 | 默认声 | 所有声 | 所有声 |
| 续集功能 | ❌ | ✅ | ✅ |
| 识字模式 | ✅(核心体验,不锁) | ✅ | ✅ |

**设计考虑**:
- **免费 6 本一次性**(非每月 3 本):让用户有一次"享受完整体验"的机会,转化质量更高
- **识字模式不锁**:免费用户也要让他看到产品的"教育价值",不然家长不买单
- **PDF 锁定**:这是高附加价值的差异化功能,必须付费

### 10.2 速率限制(防刷)

| 限制 | 数值 | 原因 |
|---|---|---|
| 免费用户每设备每天 | 2 次生成 | 防止分享账号刷 |
| 订阅用户每设备每天 | 10 次生成 | 防止 API 滥用 |
| 同一 IP 每小时 | 50 次 TTS 请求 | 防机器人 |
| 单次故事生成超时 | 180 秒 | 超过自动失败,不计费 |

### 10.3 设备离线处理

- 设备断网 < 2 小时:不影响体验,使用本地缓存的已生成故事
- 断网 > 2 小时:新生成的故事请求排队,恢复联网后自动上传
- 断网 > 7 天:设备标记为"不活跃",经销商 Console 显示警告

### 10.4 账号与设备绑定规则

#### 核心事实
- **APP 预装在硬件里**:硬件出厂即默认激活(预置激活码 + OEM 配置)
- **激活 ≠ 用户绑定**:预装代表设备可用,但必须扫码注册后才能和家长账户关联
- **绑定 ≠ 权益重置**:设备的 6 次免费生成额度属于**设备**,不属于家长

#### 绑定规则

- 一个家长最多绑定 **4 台设备**(多孩子家庭 / 客厅+卧室 / 爷爷奶奶家)
- 一台设备最多绑定 **1 个家长**(同一时刻)
- 家长可主动"解绑"设备(转让 / 送人 / 卖二手)

#### 转让场景

```
家长 A 解绑设备 → 设备状态:"已解绑"
  ↓
家长 B 收到设备,开机 → 扫码进入 H5 注册流程
  ↓
H5 检测到设备曾绑定过 → 显示"此设备由他人使用过"
  ↓
家长 B 注册 → 绑定到自己账户
  ↓
权益状态:
  - 免费生成次数(6 次):如果原主用完了就是用完了,不重置
  - 本地故事:原主的故事保留在本地缓存(可选:首次绑定时提示是否清空)
  - 订阅:属于原家长 A 账户,和设备无关,不迁移
  - 家长 B 想生成新故事 → 必须订阅(或购买单本)
```

#### 激活码状态(经销商 Console 视角)

| 状态 | 触发条件 |
|---|---|
| **未激活(未出厂)** | 刚生成,未预装到设备 |
| **已预装(未使用)** | 硬件已出厂,用户未开机 |
| **已激活(已绑定)** | 用户扫码注册,已绑定家长账户 |
| **已转让** | 原绑定家长解绑后,尚未被新家长绑定 |
| **已作废** | 设备损坏/召回,平台管理员手动作废 |

**关键**:激活码一旦用于出厂预装,**就不会回到"未激活"状态**,只会在"已激活" ↔ "已转让"之间变化。

## 十一、内容创作规范(非常重要)

### 11.1 故事文案风格

**必须**:
- 温暖、正面、有诗意(不是干瘪的句子)
- 有画面感(Sensory: 看到的/听到的/感受到的)
- 年龄适配的词汇和比喻
- 结尾让孩子有"安心感"(不以悬念收尾)

**禁止**:
- 暴力(打斗/武器/流血)
- 死亡(除非是童话里"永远的朋友"这种积极转译)
- 孤独感(故事主角不能一直是一个人)
- 恐惧元素(怪物、鬼、黑暗)—— 除非转化为"神秘朋友"
- 宗教内容
- 政治内容
- 广告植入

### 11.2 图像生成规范(水彩风格锁定)

**固定风格前缀**(所有图的 imagePrompt 都加):
```
soft Japanese watercolor illustration style, warm pastel palette,
hand-painted brushstroke texture, storybook charm
```

**角色一致性**:
- 第 1 页必须定型主角(用 OpenAI gpt-image-1.5 的 `images.edit` 模式,以 `bear_idle.png` 为参考)
- 第 2-12 页传第 1 页 URL 作为 image_url,Flux 自动继承风格

**构图规则**:
- 所有故事图 16:9 横版
- 主体在画面中心 70% 范围内,留安全边距
- 720p 投影适配:不依赖小细节,大色块为主

### 11.3 分语言文案翻译原则

- **不是直译**:按目标语言的文化习惯意译
- **专有名词保留**:WonderBear 在所有语言不变,孩子名字不翻
- **情感词优先本地化**:中文"温暖"→波兰语可能译为"przytulny"(cozy)更自然
- **双语显示时**:主语言在上(大字),副语言在下(小字斜体)
